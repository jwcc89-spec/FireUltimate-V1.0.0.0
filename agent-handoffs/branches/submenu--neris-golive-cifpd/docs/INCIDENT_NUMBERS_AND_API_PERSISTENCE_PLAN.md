## Part 1: The Three Types of “Incident Numbers” (and the future fourth)

There are **three** identifiers today (and a **fourth** once we add the API). We use these names in the plan:

| Name in this doc | Where you see it | Who sets it | Used for |
|------------------|------------------|-------------|----------|
| **1. incident_id** (future) | **Top left** on incident detail and NERIS report form (after API is added) | **Server** (API) when an incident is created | Unique key for the incident in the database and app; used in URLs and list. Replaces the current “call number” once we persist via API. |
| **2. incident_number** | **Create Incident** (“Incident Number”); **Incident Detail**; **NERIS form** Core section as “Incident internal id” | **User** (optional but **must remain available** for records) | Your own reference number (e.g. from CAD). Sent to NERIS as `incident_internal_id`. |
| **3. Incident NERIS ID** | **NERIS report form** (Core section) and **View Exports** after a report has been exported | **NERIS** (after first successful export) | NERIS’s own id for this incident (e.g. `FD17075450\|2026-Test\|1771437852`). Empty until you export; then we store it and show it. **We keep this.** |

**Current behavior (before API):** The **top left** shows the **call number** (app-generated `I-YYYYMMDD-HHMMSS`), which is the in-browser key. Once we add the API, the top left will show **incident_id** (server-generated) instead.

## Part 2: Chosen Approach — Option A + incident_id / incident_number Names

**We are using Option A:** server-assigned incident id, and **keep** the optional **incident_number** field (user-generated, required for records). We also **keep** the **Incident NERIS ID** (generated by NERIS after export).

**Names:**

- **incident_id** — Server-generated. The main unique id for the incident (database and app). Shown in top left and used in URLs and list.
- **incident_number** — User-generated. Optional field that **must remain available** for your records (e.g. from CAD). Sent to NERIS as `incident_internal_id`.
- **Incident NERIS ID** — Generated by NERIS after a successful export. Read-only in our app; we store and display it.

The rest of this plan uses these names. The schema and API will use **incident_id** for the server-generated primary key and **incident_number** for the user’s optional reference.
